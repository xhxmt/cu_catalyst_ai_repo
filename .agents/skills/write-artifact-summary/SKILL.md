# When to use
Use this skill after a task is implemented and verified, especially before review or merge.

# Required output
Write a concise summary containing:
1. what changed
2. changed files
3. commands run
4. test results
5. attached artifacts such as screenshots or walkthroughs
6. remaining risks or follow-up items

# Rules
- Be factual and specific
- Mention only commands actually run
- Separate completed verification from remaining uncertainty
- Keep the summary easy for a reviewer to scan
